# WaveStream Security Audit Notes

WaveStream runs a dependency audit in CI with:

```bash
pnpm run audit
```

The script runs `pnpm audit --audit-level moderate --json`, filters the report in `scripts/audit-security.mjs`, and intentionally ignores one documented advisory:

- `GHSA-w5hq-g745-h8pq`: `uuid` buffer bounds check in `v3`, `v5`, and `v6`.

## Current Exception

The remaining advisory is pulled through `typeorm > uuid@11.1.1`.

This project tried the direct audit recommendation, `uuid >= 14.0.0`, as a root override. That removed the advisory but broke Jest/TypeORM compatibility because `uuid@14` is ESM-only while the current TypeORM package still loads UUID through CommonJS paths. Runtime and tests must stay stable, so the override is not used.

The project does not call `uuid` `v3`, `v5`, or `v6` directly. Application identifiers are generated by PostgreSQL `uuid_generate_v4()` through TypeORM entities and migrations. The practical exposure is therefore limited to TypeORM internals until TypeORM ships a compatible patched dependency path.

## Review Policy

Revisit this exception when any of the following changes:

- TypeORM releases a version that supports `uuid >= 14` without CommonJS/Jest breakage.
- `uuid` publishes a CommonJS-compatible patched release for the affected API.
- WaveStream starts calling `uuid` directly in application code.
- A higher-severity advisory replaces or expands `GHSA-w5hq-g745-h8pq`.

Before removing the ignore, run the full gate:

```bash
pnpm run audit
pnpm lint
pnpm typecheck
pnpm test
pnpm build
pnpm test:e2e
pnpm smoke:docker
```
